`context.md` を読み込み、カスタマージャーニーマップを作成してください。
`output/` 内のペルソナ分析があれば、主要ペルソナに基づいて作成すること。

## 分析内容

以下の7段階それぞれについて分析する。

### ジャーニーステージ

1. **認知（Awareness）**: プロダクトの存在を知る
2. **検討（Consideration）**: 比較・情報収集する
3. **決定（Decision）**: 購入/契約を決める
4. **オンボーディング（Onboarding）**: 最初の7日間の体験
5. **活用（Engagement）**: 継続利用と価値の実感
6. **推奨（Loyalty）**: 他者に薦める
7. **離脱（Churn）**: 利用をやめる

### 各ステージで記述する項目

- **顧客の行動**: 何をしているか
- **思考・感情**: 何を考え、どう感じているか
- **タッチポイント**: 接触するチャネルや媒体
- **ペインポイント**: 摩擦や不満が生じるポイント
- **機会**: 顧客を喜ばせられるポイント
- **重要KPI**: このステージで追跡すべき指標
- **施策提案**: 改善のための具体的な打ち手

### 感情カーブ

各ステージの顧客満足度/感情をテキストで表現:
```
認知 → 検討 → 決定 → オンボーディング → 活用 → 推奨
 😊     🤔      😰       😃            😊      🤩
（興味） （迷い） （不安） （期待を超える） （定着） （感動）
```
のような形で、感情の起伏と対応策を可視化する。

### 重要な「真実の瞬間」（Moments of Truth）

ジャーニー全体を通じて、成否を分ける決定的な瞬間を3〜5個特定し、各瞬間での最適な対応を提案する。
