`context.md` を読み込み、リスク評価とシナリオプランニングを実行してください。
`output/` 内の既存分析をすべて参照し、他の分析で浮かび上がったリスクも取り込むこと。

## 分析内容

### 1. リスク特定（15項目）

以下のカテゴリからリスクを洗い出す:

- **市場リスク**（3〜4件）: 需要変動、競合激化、価格破壊など
- **オペレーションリスク**（3〜4件）: 人材、技術障害、サプライチェーンなど
- **財務リスク**（2〜3件）: キャッシュフロー、資金調達、為替など
- **規制リスク**（2〜3件）: 法改正、コンプライアンス、訴訟など
- **レピュテーションリスク**（2〜3件）: PR危機、データ漏洩、顧客トラブルなど

### 2. リスク評価マトリクス

各リスクについて:

| 項目 | 内容 |
|------|------|
| 発生確率 | 1（極低）〜 5（極高） |
| 影響度 | 1（軽微）〜 5（致命的） |
| リスクスコア | 確率 × 影響度 |
| 早期警戒シグナル | このリスクが顕在化する前兆 |
| 予防策 | リスク発生を防ぐための施策 |
| 緊急対応策 | リスクが顕在化した場合のアクションプラン |

スコア15以上を「要最優先対応」、10〜14を「要注意」、9以下を「モニタリング」に分類。

### 3. シナリオプランニング

**ベストケース**: すべてがうまくいった場合
- 条件、売上への影響、実現確率

**ベースケース**: 最も蓋然性の高い展開
- 条件、売上への影響

**ワーストケース**: 複数のリスクが同時に顕在化
- 条件、売上への影響、生存戦略

**ブラックスワン**: 予測困難だが起きたら根底から変わるイベント
- 想定シナリオ、対応の考え方

各シナリオについて:
- 12ヶ月後の状態を具体的に描写
- 財務影響の概算（財務分析があれば連動）
- 戦略的な対応方針

## 出力

結果を `output/10-risk.md` に保存してください。
